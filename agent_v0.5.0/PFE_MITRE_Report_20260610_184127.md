**Where Transferred:** Removable USB storage device (Drive E:), labeled as 'Authorized USB'.

**Incident Narrative:**
On March 23, 2015, the suspect, an internal employee referred to as 'Mr. Informant', engaged in unauthorized data exfiltration. Forensic analysis of the workstation 'cfreds_2015_data_leakage_pc' revealed the creation of two LNK files within the user's 'Recent' folder. These files, '[secret_project]_design_concept.LNK' and '[secret_project]_proposal.LNK', were generated at 18:38:23 and 18:37:54 respectively, and pointed directly to sensitive project data stored on an external drive mapped as 'E:\RM#1'. The use of this removable media constitutes a direct violation of corporate security policy, which prohibits the use of unauthorized storage devices for handling confidential information. The suspect attempted to mask the activity by labeling the device as 'Authorized USB', a tactic identified in the incident playbook as insufficient for authorization. This activity is consistent with the broader 'Data Leakage Case' scenario where the suspect systematically prepared and exfiltrated proprietary technology data to a rival entity.

## 7. NIST SP 800-61r2 Alignment & Containment Recommendations
**Incident Lifecycle Phase:** Detection & Analysis -> Transitioning to Containment, Eradication, & Recovery

### Immediate Containment Actions
* Immediately revoke the user's network and system access credentials.
* Physically seize and secure the workstation and the identified removable storage device (RM#1).
* Disable all USB ports on the affected workstation via Group Policy or physical port blockers.
* Initiate a forensic hold on the user's account and associated cloud storage logs.

### Eradication & Recovery
* Perform a full forensic wipe and re-image of the compromised workstation.
* Conduct a comprehensive audit of all removable media usage logs to identify other potential exfiltration events.
* Purge all unauthorized LNK files and temporary artifacts from the user's profile directory.
* Reset all passwords and API keys associated with the user's account.

### Post-Incident Activity
* Implement strict hardware-based USB whitelisting (VID/PID/Serial) to prevent the use of unauthorized devices.
* Enhance DLP policies to block file copy operations to removable media for non-authorized users.
* Deploy endpoint monitoring to alert on the creation of LNK files pointing to external or removable drives.
* Conduct mandatory security awareness training focusing on the risks of insider threats and the proper handling of sensitive project data.
