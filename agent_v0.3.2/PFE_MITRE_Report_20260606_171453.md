## 6. Executive Incident Synthesis
**Who Attributed:** Iaman Informant (Internal Employee)

**What Happened:** The suspect, a manager at a technology company, exfiltrated sensitive corporate data related to a 'secret project' to a rival company. Stolen data includes project proposals, design concepts, pricing decisions, and meeting documentation.

**Where Transferred:** Personal cloud storage and unauthorized external storage devices (USB memory stick and CD).

**How Stolen:** The suspect accessed sensitive files on the corporate workstation, created shortcuts (LNK files) to track access, and attempted to move data to unauthorized physical media. Anti-forensic techniques, including renaming files with different extensions (e.g., .jpg, .mp3) and using cloud storage, were employed to hide the activity.

**When Occurred:** March 23, 2015, to March 25, 2015.

**Incident Narrative:**
The incident involved an internal threat actor, identified as 'Iaman Informant,' who systematically exfiltrated proprietary technology data. Between March 23 and March 25, 2015, the suspect accessed multiple sensitive documents, including project proposals and pricing strategies, as evidenced by the creation of LNK files in the user's recent items directory. The suspect coordinated the theft with an external conspirator via email and utilized personal cloud storage to transmit initial samples. To evade detection, the suspect attempted to mask the data by renaming files and using unauthorized physical storage devices. The activity was ultimately identified when the suspect was intercepted at the company's security checkpoint while attempting to remove the storage media from the premises. Forensic analysis of the workstation confirmed the unauthorized access and the intent to leak corporate intellectual property.

## 7. NIST SP 800-61r2 Alignment & Containment Recommendations
**Incident Lifecycle Phase:** Detection & Analysis -> Transitioning to Containment, Eradication, & Recovery

### Immediate Containment Actions
* Immediately revoke the suspect's network and system access credentials.
* Physically isolate the suspect's workstation to prevent further data tampering or remote wiping.
* Implement a temporary block on all personal cloud storage domains at the network perimeter.
* Enforce a strict 'no-device' policy at all security checkpoints and audit existing physical access logs.

### Eradication & Recovery
* Perform a full forensic wipe and reimage of the compromised workstation.
* Conduct a comprehensive audit of all file access logs to identify any additional unauthorized data movement.
* Purge all identified malicious LNK artifacts and renamed files from the system.
* Reset all corporate passwords for the affected user account and any shared resources accessed by the suspect.

### Post-Incident Activity
* Enhance Data Loss Prevention (DLP) telemetry to alert on the creation of LNK files for sensitive document types.
* Integrate threat intelligence regarding insider threat patterns into the Security Operations Center (SOC) monitoring rules.
* Conduct mandatory insider threat awareness training for all employees with access to sensitive project data.
* Implement stricter endpoint controls to prevent the mounting of unauthorized USB and optical storage devices.
